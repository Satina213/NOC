First we're using HTML, so at the top of our index.html we need the standard boilerplate like <!DOCTYPE html>, along with the usual <html>, <head>, and <body> structure. This sets up the document so the browser knows how to interpret everything correctly.>

We’re also using the p5.js library, so we include it with a <script> tag in the <head>:

<script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.4.0/p5.js"></script>

This loads all the p5 functions like createCanvas(), draw(), and background() so we can use them in our sketches.

Random Walkers

First up is the classic random walker. Each frame (inside draw()), the walker updates its position by randomly choosing to move left/right (x-axis) or up/down (y-axis). This is typically done by generating a random number and mapping it to directional steps like +1 or -1.

This next walker uses a Gaussian (normal or Bell Curve) distribution instead of uniform randomness. Rather than always stepping exactly one pixel, we use something like randomGaussian() to determine the step size. With a mean of 0 and a standard deviation of 2, most movements are small (around ±2), occasionally larger (±4), and rarely even bigger. This creates more natural-looking motion compared to purely uniform steps.

The next version uses Perlin noise. Instead of generating completely independent random values each frame, we sample from noise(), which produces smoothly changing values over time. That means the direction isn’t truly random frame-to-frame—it’s correlated with the previous frame. The result is motion that feels more fluid and continuous rather than jittery.

This is another Perlin noise example, likely exploring how changing the input (like incrementing a time variable) affects the output. Small changes to the input of noise() result in small changes in output, which is what gives that smooth transition effect.

This sketch uses 2D Perlin noise to determine the color of every pixel. By sampling noise(x, y) across the canvas, each pixel gets a value that’s similar to its neighbors, creating that smooth, cloud-like texture. It looks random, but it’s actually highly structured randomness—which is why it feels so natural.

This uses 3D Perlin noise in order to accomplish animation. Each frame of animation is like a slice of swiss cheese and the next frame is the next slice. So over time you'll see clouds come in and fade out similiar to the holes in the cheese.
